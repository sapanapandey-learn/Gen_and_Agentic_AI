📈 Line Plot — plt.plot()
Things that change over time or sequence
Healthy training curve
Overfitting curve
plt.plot(train_losses, label='Train', color='blue')
plt.plot(val_losses, label='Val', color='red')
plt.xlabel('Epoch') # x axis label
plt.ylabel('Loss') # y axis label
plt.legend() # show colour legend
HOW TO READ IT — PATTERNS
✅ Both lines going down
Model is learning. Normal behaviour. Keep training.
✅ Small gap between train and val
Model generalises well. Not memorising training data.
❌ Val loss goes UP while train keeps going DOWN
Overfitting. Model memorising training data. Add Dropout, get more data, stop training earlier.
❌ Both lines flat from start
Model not learning at all. Learning rate too low, wrong architecture, bug in code.
⚠️ Loss bouncing up and down
Learning rate too high. Reduce it. Model is overshooting the minimum.
Where used in MLTraining/validation loss curves. Accuracy over epochs. Learning rate vs loss. Any metric tracked during training.
📊 Bar Chart — plt.bar()
Comparing values across categories
Feature importance
Class distribution (imbalance check)
plt.bar(categories, values, color='steelblue')
plt.barh(categories, values) # horizontal bars
plt.xticks(rotation=45) # tilt x labels if crowded
HOW TO READ IT — PATTERNS
✅ Feature importance bar chart
Tall bars = important features. Short bars = can be dropped. Helps with feature selection — remove near-zero bars to simplify model.
❌ Class imbalance — one bar much taller
If "Class 0" bar is 850 and "Class 1" bar is 150 — your model will be biased. Need to oversample minority class or use class weights.
⚠️ Model comparison bar chart
Compare accuracy of different models side by side. The tallest bar = best model. But also check if differences are statistically significant.
Where used in MLFeature importance (Random Forest, XGBoost). Class distribution before training. Comparing model accuracies. Category counts from value_counts().
🔵 Scatter Plot — plt.scatter()
Relationship between two numeric variables
Strong positive correlation
Two separable classes
plt.scatter(x, y, color='blue', alpha=0.5)
# alpha = transparency (useful for overlapping dots)
plt.scatter(x, y, c=labels, cmap='RdYlGn') # colour by class
HOW TO READ IT — PATTERNS
✅ Dots forming a diagonal line (up-right)
Strong positive correlation. As X increases, Y increases. These two features move together — consider dropping one to avoid redundancy.
✅ Two clear clusters of different colours
Classes are linearly separable. A simple model (logistic regression) might work. No need for deep network.
❌ Colours completely mixed together
Classes overlap heavily. These two features alone can't separate them. Need more features, or a non-linear model.
⚠️ One dot far from all others
Outlier. Could be data entry error. Could skew your model heavily. Investigate before training.
Where used in MLChecking feature correlations before training. Visualising clusters (K-Means results). Actual vs predicted values in regression. PCA/t-SNE dimensionality reduction output.
📉 Histogram — plt.hist()
Distribution of one variable — how values are spread
Normal distribution
Skewed distribution
plt.hist(df['age'], bins=20, color='steelblue', edgecolor='black')
# bins = how many buckets to divide data into
# edgecolor = outline on each bar so they're visible
HOW TO READ IT — PATTERNS
✅ Bell curve shape (normal distribution)
Values clustered around the middle. Most ML algorithms work best with this. If you see this — good, no preprocessing needed.
❌ Long tail to the right (right skew)
Salary, house prices often look like this. A few very high values pull the mean up. Apply log transformation before training: np.log(df['salary'])
❌ Two separate humps (bimodal)
Two different groups mixed in one column. Male/female heights in one histogram. Consider splitting into separate features or groups.
⚠️ All values crammed in one bucket
Feature has almost no variance. Useless for a model — drop it. Or increase bins to see finer detail.
Where used in MLChecking feature distributions before training. Spotting skewed features that need log transform. Finding outliers. Checking if normalisation/standardisation is needed.
📦 Box Plot — plt.boxplot()
Shows spread, median, and outliers of a variable
Anatomy of a box plot
plt.boxplot(df['salary'])
plt.boxplot([group1, group2, group3]) # compare multiple groups
HOW TO READ IT — PATTERNS
📦 The box itself
Middle 50% of your data. The "IQR" (interquartile range). Bottom of box = 25th percentile. Top = 75th percentile. Wider box = more spread in the middle.
━ Line inside the box
The median (middle value). If it's near the centre of the box → balanced. If it's near the top or bottom → skewed data.
│ Whiskers (lines above/below box)
Extend to 1.5× the box size. Shows the expected range of values.
❌ Dots beyond the whiskers
Outliers. Values far outside normal range. Each dot is one data point that's unusually extreme. Investigate — could be data errors or genuinely rare cases.
✅ Comparing box plots side by side
Compare salary of churned vs not-churned customers. If boxes don't overlap much → salary predicts churn well. If completely overlapping → salary is useless as a feature.
Where used in MLOutlier detection before training. Comparing feature distributions across classes. Checking if classes have different distributions (feature usefulness check).
🌡️ Heatmap — sns.heatmap()
Colour grid showing correlation between all features
Correlation matrix heatmap
import seaborn as sns
corr = df.corr() # compute correlations
sns.heatmap(corr,
    annot=True, # show numbers in cells
    cmap='coolwarm', # red=positive, blue=negative
    fmt='.2f') # 2 decimal places
HOW TO READ IT
🔴 Dark red cell = +1.0
Perfect positive correlation. As one goes up, other goes up. Drop one — they're redundant.
🔵 Dark blue cell = −1.0
Perfect negative correlation. As one goes up, other goes down. Also redundant — drop one.
⬜ White/light cell = ~0
No correlation. Features are independent of each other. Both are useful — keep both.
📍 Diagonal is always 1.0
Every feature perfectly correlates with itself. Ignore the diagonal — it's meaningless.
Where used in MLFeature selection — drop highly correlated features (>0.9). Finding which features correlate with target variable. Multicollinearity check before linear models.
⚙️ Important Matplotlib Things to Remember
The things that trip up beginners
THE STANDARD TEMPLATE — use every time
plt.figure(figsize=(10, 6)) # always set size first
plt.plot(...) # draw your graph
plt.xlabel('X label') # always label axes
plt.ylabel('Y label')
plt.title('My Title') # always add title
plt.legend() # if multiple lines
plt.tight_layout() # prevents labels getting cut off
plt.show() # display the graph
MULTIPLE PLOTS SIDE BY SIDE
fig, axes = plt.subplots(1, 2, figsize=(12, 5)) # 1 row, 2 cols
axes[0].plot(train_loss) # left plot
axes[0].set_title('Loss')
axes[1].plot(accuracy) # right plot
axes[1].set_title('Accuracy')
COMMON THINGS PEOPLE FORGET
❌ Forget plt.legend()
You set label='Train Loss' but no legend appears. Always call plt.legend() after all plt.plot() calls.
❌ Labels overlapping
X-axis category names crash into each other. Fix: plt.xticks(rotation=45) to tilt them.
❌ Title/labels cut off
Graph looks cropped. Fix: add plt.tight_layout() before plt.show().
❌ Plots stacking on each other
Previous plot bleeds into next. Fix: call plt.figure() or plt.clf() before each new plot.
USEFUL STYLING OPTIONS
plt.plot(x, y, color='#a78bfa') # hex colour
plt.plot(x, y, linewidth=2) # thicker line
plt.plot(x, y, linestyle='--') # dashed line
plt.plot(x, y, marker='o') # dot at each point
plt.scatter(x, y, alpha=0.5) # 50% transparent dots
plt.scatter(x, y, s=100) # bigger dots (size)
plt.axhline(y=0.5, color='gray', linestyle='--') # horizontal reference line
plt.grid(True, alpha=0.3) # light grid lines
