🧮 The Math Behind Neural Networks

From y = mx all the way to your neural network — every piece of math, explained from scratch

① Where does y = mx come from?
The oldest idea in math: things are related to each other
Imagine you're a shopkeeper. You sell mangoes. Each mango costs ₹5.

If someone buys 3 mangoes, total = ₹15.
If someone buys 7 mangoes, total = ₹35.

You just wrote a rule: total = 5 × quantity

In math notation, that's: y = 5x. That's it. That's y = mx.
y = m · x
y = output  |  m = rate (slope)  |  x = input
m is the "rate" — how much y changes when x changes by 1.
m = 5 means "for every 1 unit of x, y goes up by 5."
Move m (slope):m = 2
Why does this matter for neural networks? A single neuron IS this formula. It multiplies input by a weight (m) to get an output. The entire network is just many of these happening at once.
② Adding bias: y = mx + b
What if things don't start at zero?
Back to mangoes. But now there's a ₹10 delivery charge regardless of how many you buy.

0 mangoes → still pay ₹10 (delivery).
3 mangoes → 5×3 + 10 = ₹25.
7 mangoes → 5×7 + 10 = ₹45.

Rule: total = 5 × quantity + 10 → y = mx + b
y = m · x + b
b = bias (the fixed starting value when x = 0)

❌ Without bias (y = mx)
Line MUST pass through origin (0,0). Even if all scores are 0, prediction is forced to be 0. Very limiting — data rarely starts at zero.
✅ With bias (y = mx + b)
Line can sit anywhere. Even when all inputs are 0, the neuron can still output a non-zero value. More flexible, fits more patterns.
The neuron in your code: z = X @ W + b is literally y = mx + b. W is m (slope/weight), b is b (bias). Your network learns both.
③ What if you have many inputs?
Extending y = mx + b from 1 feature to 3 features
y = mx + b works for one input. But your students each have 3 scores.

Now you need one weight (m) for each input:
1
One score: y = w₁·score₁ + b
2
Two scores: y = w₁·score₁ + w₂·score₂ + b
3
Three scores (your code): y = w₁·score₁ + w₂·score₂ + w₃·score₃ + b
z = w₁x₁ + w₂x₂ + w₃x₃ + b
This is called a "weighted sum" or "dot product + bias"
Each weight wᵢ says: "how much does feature i matter for the prediction?"

High w₂ = score₂ matters a lot.
Near-zero w₁ = score₁ is almost irrelevant.

The network learns these weights automatically. You start them small and random, and gradient descent finds the right values.
This is called a linear combination — adding up things multiplied by constants. It's the core operation in ALL of linear algebra and ALL of neural networks.
④ Why do we use matrices?
Pure efficiency — doing it for all students at once
You have 5 students. You could write one equation per student:
z₁ = w₁·x₁₁ + w₂·x₁₂ + w₃·x₁₃ + b  ← student 1
z₂ = w₁·x₂₁ + w₂·x₂₂ + w₃·x₂₃ + b  ← student 2
z₃ = w₁·x₃₁ + w₂·x₃₂ + w₃·x₃₃ + b  ← student 3
z₄ = w₁·x₄₁ + w₂·x₄₂ + w₃·x₄₃ + b  ← student 4
z₅ = w₁·x₅₁ + w₂·x₅₂ + w₃·x₅₃ + b  ← student 5
That's 5 separate equations. Now imagine 1 million students. Writing loops is slow.

Matrices let you do all 5 at once in one operation:
Z = X · W + b
(5×3) · (3×1) + (1,) = (5×1) — all 5 students, one operation
How matrix multiply works for one student (row × column):
[x₁ x₂ x₃]  ·  [w₁]
                 [w₂]  =  x₁·w₁ + x₂·w₂ + x₃·w₃
                 [w₃]
The inner dimensions must match: (5×3) · (3×1). The 3s cancel. Result: (5×1).
Think of it as: each row of X is one student's data. Each row gets multiplied by W and produces one output number. All rows processed simultaneously. This is why GPUs are fast — they're built for exactly this.
⑤ The problem with pure math (why activation functions exist)
y = mx + b can only draw straight lines — but real data isn't straight
y = mx + b is a linear equation. No matter how many you stack, it's still linear.

Real problems are non-linear:
✗
Can a straight line separate cats from dogs in an image? No.
✗
Can y=mx+b output a probability (0 to 1)? No — it goes to ±infinity.
✗
Two linear layers stacked = still one linear layer. Depth becomes useless.
Solution: after the linear step, apply a non-linear function. This is called an activation function.
Linear step
z = Xw + b
→
Activation
σ(z) or ReLU(z)
→
Output
probability / class
Without activation
z = Xw + b → raw number (-∞ to +∞). Can't be a probability. Stack 100 layers — still just a line. Network can only learn linear patterns.
With activation
σ(Xw + b) → 0 to 1. Stack 2 layers → can learn curves. Stack 10 layers → can learn almost ANY pattern. This is why deep learning works.
The key insight: Linear + non-linear, repeated many times = a universal function approximator. Given enough neurons, a neural network can learn ANY pattern. The activation function is what makes this possible.
⑥ Sigmoid — where does the formula come from?
You need a function that squashes any number to (0,1). Sigmoid is the mathematical answer.
You want a function f(x) such that:
• f(-∞) = 0
• f(0) = 0.5
• f(+∞) = 1
• Smooth curve everywhere (differentiable — needed for gradients)

The natural mathematical function that does this is:
σ(z) = 1 / (1 + e⁻ᶻ)
e = Euler's number ≈ 2.718 (the base of natural logarithm)
Where does e come from?

e is a special number (≈2.718) that appears naturally in growth, decay, and probability.
e⁻ᶻ means "e to the power of negative z".

When z = 0:   e⁰ = 1, so σ(0) = 1/(1+1) = 0.5
When z = +big: e⁻ᶻ → 0, so σ = 1/(1+0) = 1
When z = -big: e⁻ᶻ → ∞, so σ = 1/(1+∞) = 0
You don't invent this formula. Mathematicians derived it from probability theory (logistic regression was invented in 1958). You use it because it's the proven answer to "how do I get a smooth 0-to-1 output?" For deep hidden layers today, ReLU is preferred because sigmoid has the vanishing gradient problem — but for the output layer in binary classification, sigmoid is still correct.
⑦ Loss Function — where does it come from?
Not invented arbitrarily — derived from information theory and probability
After sigmoid you have a prediction ŷ (say 0.8 = "80% chance of passing").
True label y = 1 (they did pass).

You need ONE number that answers: "how wrong am I?"

Why not just use (ŷ − y)²? (Mean Squared Error)
You could. But when predictions are near 0 or 1, MSE has very small gradients → slow learning. Cross-entropy has better gradient properties.
L = −[y·log(ŷ) + (1−y)·log(1−ŷ)]
Binary Cross-Entropy — from Maximum Likelihood Estimation in statistics
Where does log come from?

log is the inverse of e. If e² = 7.38, then log(7.38) = 2.
Key property: log(1) = 0, log(small number) = very negative.

So −log(ŷ) when ŷ=0.99 → −log(0.99) ≈ 0.01 (correct, small loss)
So −log(ŷ) when ŷ=0.01 → −log(0.01) ≈ 4.6 (very wrong, big loss)

This is what you want — punish confident wrong answers heavily.
y=1
Formula becomes: −log(ŷ). Loss is 0 when ŷ=1 (perfect), explodes when ŷ→0.
y=0
Formula becomes: −log(1−ŷ). Loss is 0 when ŷ=0 (perfect), explodes when ŷ→1.
Origin: This comes from Maximum Likelihood Estimation — a statistics concept from 1920s. You're asking "what weights make the observed data most probable?" Maximising likelihood = minimising cross-entropy loss. Same thing, different language.
⑧ Gradient — what is it actually?
From calculus: the derivative tells you the slope at a point
You learned y = mx in school. The derivative of y with respect to x is m — the slope.

For a curved function like loss L, the derivative at any point tells you:
"If I increase W by a tiny amount, does L go up or down, and by how much?"

This is written as ∂L/∂W (partial derivative of L with respect to W).
dW = ∂L/∂W = (1/n) · Xᵀ · (ŷ − y)
Derived by applying chain rule through: loss → sigmoid → linear layer
Chain rule (you don't need to derive it, just understand it):

Loss depends on ŷ. ŷ depends on z. z depends on W.
To find how loss changes with W, you multiply the rates through:
∂L/∂W = (∂L/∂ŷ) × (∂ŷ/∂z) × (∂z/∂W)

When you work this out mathematically for BCE + sigmoid + linear, it beautifully simplifies to just: (ŷ − y) × X.

This is why the gradient formula looks simple — the math cancels out cleanly.
Positive gradient (dW > 0)
Loss increases when W increases. So decrease W. Move opposite direction: W = W − α·dW
Negative gradient (dW < 0)
Loss decreases when W increases. So increase W. Moving opposite still works: W = W − α·(negative) = W + something
Always subtract the gradient. Gradient points toward steepest increase in loss. You want to decrease loss. So move opposite = subtract. This is gradient descent.
⑨ The Full Picture — how it all connects
Every piece of math you just learned, in one chain
1
y = mx + b → z = XW + b
Weighted sum. Each weight says "how important is this feature?" Bias allows offset. Pure linear algebra.
2
Activation → ŷ = σ(z) = 1/(1+e⁻ᶻ)
Squash to 0–1. Makes output a probability. Non-linearity allows network to learn curves, not just lines.
3
Loss → L = −[y·log(ŷ) + (1−y)·log(1−ŷ)]
Measures wrongness. From statistics (MLE). log punishes confident wrong predictions heavily.
4
Gradient → dW = (1/n)·Xᵀ·(ŷ−y)
Derivative of loss w.r.t. weights. Chain rule through all layers. Tells you which direction to move.
5
Update → W = W − α·dW
Move weights opposite to gradient. α (learning rate) controls step size. Repeat 1000+ times = training.
z=XW+b → σ(z) → L → dW → W_new → 🔁
The math fields behind this:
• Linear Algebra → z = XW + b (matrix operations)
• Calculus → derivatives, chain rule, gradient
• Statistics / Probability → sigmoid, cross-entropy loss, MLE
• Optimisation → gradient descent, learning rate

You don't need to master all four first. Learn them alongside the neural network — the network gives you the "why" for each concept.
The big answer to your question: These aren't arbitrary decisions. Every formula was derived from math. Sigmoid → from probability theory. Cross-entropy → from statistics. Gradient → from calculus. You're not supposed to invent these — you learn which formula fits which problem and why it was derived.
